## How these 10 were chosen (selection criteria)

This is **not** a list of "first Google result" influencers. Every person here was verified live (real
profile, current role, content dated 2024–2026) and had to pass three filters:

1. **They practice what they teach** — each one runs an agency, builds a tool, or sells the service.
   Not commentators; operators with skin in the game and real send/call volume.
2. **They cover a distinct part of the pipeline** — the set is deliberately spread across copywriting →
   deliverability → data/AI → multichannel → cold calling → agency ops, so it can support a *complete*
   playbook rather than 10 people repeating one idea.
3. **Insiders respect them** — these are the names that practitioners actually cite, not SEO-bait.

## Collection methodology & honest limitations

- **YouTube transcripts** — pulled programmatically with `yt-dlp` (to resolve channels + list recent and
  topical uploads) and `youtube-transcript-api` (to fetch caption tracks). 27 transcripts captured before
  hitting YouTube's IP rate limit (HTTP 429); the remainder are documented per-folder.
- **Newsletters / blogs** — scraped with Firecrawl (markdown, main-content only). Each file carries a
  header with author, source URL, publish date, and source type.
- **LinkedIn** — Firecrawl (and most automated scrapers) are blocked at the provider level for
  `linkedin.com` profile/activity URLs. Per the brief, which permits manual collection, each author's
  `linkedin-posts/<author>/posts.md` contains: the profile URL, genuine post excerpts captured via web
  search (with source links), and a clear manual-collection checklist. (Exception: LinkedIn `/pulse/`
  article URLs *do* scrape — used to capture Leslie Venetz's newsletter pieces.)
- **No fabrication** — only real, retrieved content was saved. Where a source could not be fetched
  cleanly (e.g. Will Allred's SPA-routed Lavender articles, paywalled newsletter editions), that is
  stated explicitly in the relevant file rather than filled with invented text.
